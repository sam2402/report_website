Design Principles

Before designing the optimal User Interface (UI) for our dashboard, we looked at the design principles to gain a better understanding of the key features that are essential:


User-Centered Design: While creating a user interface, we must consider the needs and goals of the end users. UCD involves comprehending the context, tasks, and objectives of the user before developing the dashboard. For instance, if a sales infrastructure manager is the dashboard's key user, the dashboard should show data that aids in their decision-making about server CPU usage and electricity consumption. This makes UCD important since the dashboard created is user-friendly, intuitive, and satisfies the userâ€™s needs.

Minimalism: The design should be limited to the absolute necessities. To prevent overwhelming the user with information, this involves just presenting the most necessary data. For instance, the future advice section in the dashboard can display just the categories of advice and enable users to drill down for more information through the use of an accordion rather than listing every single future advice to reduce carbon emissions. It helps users focus on the most crucial information and lessens cognitive overload.

Visual Hierarchy: This technique involves employing visual cues like colour, size, and contrast to draw the user's attention to the most important content. For instance, to make the most crucial information on the dashboard stand out, the font size or colour could be increased. By focusing the user's attention on the most crucial information, visual hierarchy aids in quick and simple understanding of the material.

Feedback: Feedback is necessary to let users know that their actions have been taken into account and to ensure that the dashboard is functioning effectively. A dashboard button may show a message or change colour for a brief second when the user clicks it to indicate that the activity was successful. Feedback is crucial since it clarifies how to utilise the dashboard for the user and shows that their actions are having the desired impact.

Flexibility: This involves creating a dashboard that is adaptable enough to adapt to the users needs. The dashboard might, for instance, give users the option to alter the information displayed or select different visualisations. Building a sidebar and enabling page switching can accomplish this. Flexibility is crucial because it guarantees that the dashboard may be tailored to the needs and preferences of the user, increasing its use cases for multiple users.

Consistency: The design, font, and colour scheme must all be consistent throughout the dashboard. This makes it easier for the user to navigate and understand how the various components relate to one another. The user will know which buttons are clickable on the dashboard if the buttons are all the same colour and style. Consistency is crucial since it makes the user familiar with the dashboard, making it simpler to operate and avoid confusion.
