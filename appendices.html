User Manual

Welcome to our Carbon Dashboard! This user manual will guide you through every functionality of our dashboard, with a troubleshooting guide for some common issues. Hope you enjoy using our dashboard :)  

 

Pages 

You can browse all the information by looking up the navigation bar on the left. Here is the introduction for each page.  


  • Welcome 

    • In this page, we provide a map to give information about the current global carbon emissions. You can have a basic understanding of the current emissions in each region and the differences between the different regions. 

     

  • Past Usage 
    •  

    This page illustrates the carbon mission in the last 30 days, consisting of four main components: 

     

      1. Resource Group Bar 

      • Select to display data from different resource groups. 

     


      2. Total Carbon Usage

      • Shows carbon emission data in the past 30 days in different regions. 

      • Click the regions labels on the left to show / hide its data.  

      • Hover over the point in lines to see the concrete values of the emission.  

     

      3. Emissions type per region

      • Shows the sources of carbon emissions in different regions from last 30 days.  

      • Hover over the bar to see the amount of emission from that source.  

     

      4. Emission breakdown

      • Shows the proportion of total carbon emissions in terms of renewability.  

     

  • Future Prediction 

    • This page shows the prediction carbon mission for the next 3 days.  

     

      1. Resource Group Bar

      • Select to display data from different resource groups. 

     

      2. Future Emission by Resource / Location

      • Show the predicted carbon emissions for different resources / locations.  

     

  • Resource Group / Region 

    • These pages give advice for reducing carbon emissions for distinct resource groups / regions.  

     

      1. Resource Group Bar

      • Select to display advice for different resource groups. 

     

      2. Advice Boxes

      • Show the advice according to energy type, location, resources configuration and cooling type.  

      • Click the title for each piece of advice to hide / show its advice.  

     

     

    Functional Buttons 

    There are a few functional buttons provided on the sidebar & top bar. Here is the introduction of their functions.  

     

      1. Menu Button

      • In the top left corner of the dashboard, there is a menu button “≡”. Tapping on it to hide/show the sidebar.  

     

      2. Theme Button

      • In the top right corner of the dashboard, there is a theme button “☼” / “☾”. Click it to change the dashboard theme to light / dark mode. Notice that the default theme follows system settings.  

    Troubleshooting Guides

    You may encounter some issues when using our dashboard, here are some solutions to the common problems.

     

      Why there is no data displayed in data pages? 

     

      • It will take a few seconds to fetch data from azure every time you start the dashboard. Once it is done, the data will be shown properly. And due to our caching mechanism, subsequent data are updated immediately, so this issue won’t happen again. 

     

      Why does the advice still not shown up after a while? 

     

      • Sorry for the long waiting time. Since our advice system is supported by GPT-3 from OpenAI, it usually takes a minute to generate the advice. It may take longer if the GPT-3 server is in peak usage.  

     

    Not in the list?  Feel free to contact the development team.

    Deployment Manual

    1. Clone this repository

    2. Follow the instructions here to install the Azure CLI

      How to install the Azure CLI

    3. Once this is installed, open your command line and enter

      az login

      Following the instructions to login to your Microsoft account.

      If you are Adamos Hadjivasiliou, Yun Fu or Dean Mohamedally, you should have been given access to our system by our client Avanade. If you encounter errors when logging in or when running our dashboard please contact me at sam.lavadera.20@ucl.ac.uk.

    4. Go to the directory carbon-dashboard you created by cloning this repository. Start the server by running the following commands:

      cd backend
./server.sh

    5. After a few seconds the following will appear in your command line:

      Running on http://127.0.0.1:5000

      Go to http://127.0.0.1:5000/ in your browser

    6. Go back to your terminal in which you started the server. Wait until you see "SERVER READY!" in your terminal. This will take several minutes.

    7. Keeping the server running, open a new terminal window in the carbon-dashboard directory.

    8. Run the following

      cd frontend
npm install
npm start

    9. Open http://localhost:3000 in your browser.

    10. You will now see the carbon dashboard! Some graphs and figures may take up to a minute to load especially the first time you open a page. After this it will speed up as we have implemented a cache on the backend for common requests.
