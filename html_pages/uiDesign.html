Design Principles

Before designing the optimal User Interface (UI) for our dashboard, we looked at the design principles to gain a better understanding of the key features that are essential:


User-Centered Design: While creating a user interface, major consideration is given to the needs and goals of the end users. UCD entails comprehending the context, tasks, and objectives of the user before developing the dashboard with them in mind. For instance, if a sales infrastructure manager is the dashboard's key user, the dashboard should show data that aids in their decision-making about server CPU usage and electricity consumption. UCD makes sure the dashboard is user-friendly, intuitive, and satisfies their needs, making it crucial.

Minimalism: The design is limited to the absolute necessities. To prevent overwhelming the user with information, this entails just presenting the most necessary data. For instance, the dashboard can show a few essential parameters and enable users to drill down for more information rather than listing every single data related to carbon emissions. It helps users focus on the most crucial information and lessens cognitive overload.

Visual Hierarchy: This technique involves employing visual cues like colour, size, and contrast to draw the user's attention to the most important content. For instance, to make the most crucial information on the dashboard stand out, the font size or colour could be increased. By focusing the user's attention on the most crucial information, visual hierarchy aids in quick and simple understanding of the material.

Feedback: Feedback is necessary to let users know that their actions have been taken into account and to ensure that the dashboard is functioning effectively. A dashboard button, for instance, may show a message or change colour when the user hits it to indicate that the activity was successful. Feedback is crucial since it clarifies how to utilise the dashboard for the user and shows that their actions are having the desired impact.

Flexibility: This entails creating a dashboard that is adaptable enough to alter as the needs of the user do. The dashboard might, for instance, give users the option to alter the information displayed or select different visualisations. Building a sidebar and enabling page switching can accomplish this. Flexibility is crucial because it guarantees that the dashboard may be tailored to the needs and preferences of the user, increasing its use and efficiency.

Consistency: Design, font, and colour scheme must all be consistent throughout the dashboard. This makes it easier for the user to navigate the dashboard and understand how the various components relate to one another. The user will be aware that each clickable button on the dashboard if the buttons are all the same colour and style. Consistency is crucial since it makes the user feel at ease and familiar with the dashboard, making it simpler to operate.
